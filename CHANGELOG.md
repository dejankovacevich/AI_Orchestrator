## [0.1.0] - 2026-05-20

First tagged release. The scaffold inherited from the original Codex draft has
been turned into a working clarification + execution pipeline with a
maintainable architecture and a documented contribution path.

### Added

#### Core pipeline
- **Step-based execution graph** (`assistant_core/execution/graph.py`,
  `steps.py`, `runner.py`). Each step is a function with signature
  `StepContext -> StepContext`. Adding a step is one line in `graph.py`.
- **File classifier** routing code suffixes to `local-coder` and everything
  else to `local-main`.
- **Secondary retry chain**: when the deterministic evaluator says the
  primary local output is weak, the runner retries on `local-secondary`
  (llama 3.3 70B), promotes the retry when it passes.
- **Cloud-review gate (catalog-only by default)** at
  `cloud_review_gate`. Flagged files are written to
  `08_CLOUD_REVIEW_CANDIDATES.md` with the safety-gate verdict. Default-off;
  no cloud spend unless `cloud_fallback_enabled=true` + `ANTHROPIC_API_KEY`.
- **Task types** with auto-detection: `morning_brief`, `code_review`,
  `test_generation`, `doc_generation`, `decision_capture`, `risk_scan`. Two
  types (`morning_brief`, `code_review`) have specialized prompts today; the
  rest fall back to `morning_brief` until specialized prompts land.
- **Grounding enforcement** via `WorkPacket.grounding_required` — the
  deterministic evaluator can require `Source:` citations in every output.

#### Safety
- **Day-unlock switch** (`scripts/day_unlock.sh` / `day_lock.sh`) — file
  flag + env-var override; visible banner in the Streamlit panel.
- **`assert_model_allowed` gate** on every local model call. Fail-closed
  resolution: unmapped tags route to `local-main` and therefore through
  the DAY_MODE gate.
- **`assert_cloud_allowed` six-gate check** before any cloud call.
- **`assert_original_file_write_allowed`** and
  **`assert_no_external_write`** to keep writes inside allowed roots.

#### Storage
- Postgres schema (7 tables): work_packets, clarification_questions,
  execution_runs, model_calls, evaluations, artifacts, memory_candidates.
- Storage layer split into `assistant_core/storage/` package by entity
  (no more single 360-line module).

#### UI
- Streamlit control panel with seven tabs: Dashboard, Create Work Packet,
  Clarification, Models, Schedule, Execution, Artifacts.
- Header status strip showing mode, day unlock, time window,
  auto-execution status, and per-service up/down (Postgres, Temporal,
  Ollama, LiteLLM).
- Models tab: load / unload / quick-test with gate enforcement.
- Schedule tab: editable day/night times with atomic config writes.

#### Operations
- Docker Compose stack: Postgres 16, Temporal 1.26, Temporal UI 2.31,
  Open WebUI.
- Temporal worker + workflows + activities wired to the runner.
- `scripts/review_code.sh` — one-shot synchronous code review (no Temporal).
- `scripts/export_audit.sh` — full audit-trail JSON dump per packet.
- launchd plist generator reads `night_mode_start` from config; arming
  remains a deliberate three-step terminal gesture.

#### Documentation
- Three PDFs: System Guide (34 pages), Onboarding (25 pages),
  Technical Reference (40 pages). All generated by
  `scripts/build_pdfs.py` (split into `scripts/pdf_builders/` package).
- AGENTS.md — rules for human or AI contributors.
- ONBOARDING.md — quick reference plus 5 use-case recipes.
- docs/day_unlock.md, docs/models_panel.md, docs/auto_execution.md —
  per-feature design rationale.
- CONTRIBUTING.md, ROADMAP.md.

#### Co-work scaffolding
- GitHub Actions CI workflow (shell syntax + py_compile sweep + pytest
  + PDF smoke build + docker compose config).
- Issue templates (bug, feature) and PR template (with safety checklist).
- 6 seeded GitHub issues, 4 tagged `good-first-issue`.
- Branch protection on `main` requiring all 3 CI jobs.
- GitHub Discussions enabled.

### Testing

153 pytest tests covering: clarification flow, readiness scoring, safety
gates (parameterized over every local model group and mode), task-type
auto-detection, file classifier, secondary retry chain, cloud-review
catalog, output writers, audit logging, schema serialization, scheduler
status, config writer atomicity, Ollama admin API, day-unlock isolation,
PDF generation.

### Repository hygiene

- Username and absolute paths scrubbed from all tracked files and PDFs.
- Personal names replaced with `Scope1` placeholder in fixtures.
- MIT license with author metadata in `pyproject.toml` and `README.md`.
